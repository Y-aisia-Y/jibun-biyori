## サービス概要
日々の体調管理を行いたい方、復職を目指す方のためのWebアプリケーションです。
1日の予定を縦型のバーチカルカレンダーで、1時間単位で可視化・管理します。
記録を可視化することで、自身の体調把握や、医師・会社への報告資料作成をサポートします。

## このサービスへの思い・作りたい理由
このサービスの原点は、私自身の適応障害による休職経験です。

休養期、回復期を通過し、復職準備期に入ると睡眠時間、食事、運動や作業などの項目に沿って一日の出来事を記録するワークを始めます。
日々の気分や体調の変動を記載することで、生活リズムとメンタルの関連を可視化でき、客観的に自身を見直すことができます。
また、復職後の再発防止にも効果的と言われています。それが「生活記録表」です。

記録方法は医療機関や会社によりますが、主にExcelか手書きが一般的です。
ただ、既存のExcelや手書きでの記録は、回復過程にある身には苦痛で面倒に感じる事が多々ありました。

このサービスは、単に記録の面倒さを解消するだけでなく、「自身の回復プロセスを客観的に可視化し、未来の自分への自信に繋げる」ためのツールとして開発を決意しました。
このサービスで、少しでも復職トレーニングに取り組むハードルを下げ、記録を無理なく続け、未来への自分につなげられたらと思います。

## ユーザー層について
- 休職中で、復職を目指している方
 私自身がそうであったように、生活記録表と報告が（会社や主治医から）求められる層であり、最も「手書きやExcelでの記録は面倒」という課題を強く感じているため。このサービスの中核となるターゲットです。

- 主治医に日々の体調を報告する必要がある方（通院中の方）
 診察の際、「最近どうでしたか？」という曖昧な質問に対し、客観的な記録（睡眠時間、気分の浮き沈み、服薬状況など）を見せることで、より正確な状態を伝え、医師との円滑なコミュニケーションを助けることができるため。

- なんとなく不調が続き、自分の生活リズムを見直したい方
 深刻な休職には至っていないものの、自身の生活パターンを可視化したいというニーズを持つ層です。記録を続けることで、自身の不調のパターンを発見し、セルフケアや生活改善のきっかけとして利用してもらうことを想定しています。

## サービスの利用イメージ
- 構造化されたバーチカルカレンダーでのリズム管理
 1日の予定を1時間単位で入力できます。
 手帳感覚で直感的に予定を入力でき、生活に構造的なリズムを取り戻すことができます。

- データ化による傾向把握
 「睡眠時間」「疲労感」「意欲」「活動量」といった、数字による5段階のチェック項目を記録します。
 週次・月次のグラフで体調の傾向を把握できます。

- 回復を実感する記録の進化
 回復が進み、自身を振り返る余力が出てきた段階で、ユーザーは「今日はどんな気分だった？」など、内省的な項目を任意で追加できます。
 これにより、ネガティブな記録項目からポジティブな感情の記録へと移行するプロセスそのものを回復の証として実感し、利用を継続できます。

## ユーザーの獲得について
- 休職者・復職準備中の層
 復職・休職の体験談を共有しているアカウントやコミュニティに向けて、アプリの体験を共有。

- 通院中の層・体調を見直したい層
 「医師への報告資料作成が簡単」「体調に合わせた項目を任意で追加できる」という差別化ポイントを前面に出し、告知する。

- 継続利用の促進
 回復が進んだユーザーが、日常のシンプルな記録を継続して行えるよう、アプリ内でポジティブなフィードバックや達成感を感じられる仕掛けを導入する。

## サービスの差別化ポイント・推しポイント
- 回復度合いに応じて記録項目を追加できる
 既存の体調管理アプリが常に同じ項目を記録させ続けるのに対し、本サービスはユーザーの任意で項目の追加・削除・切り替えができます。
 ユーザーは記録のシンプル化を「回復の証」として実感でき、ネガティブな記録からポジティブな記録へ移行するプロセスを楽しむことができます。
 これにより、「続けて使いたくなる」工夫を実現します。

- 復職・体調報告に特化したバーチカルビュー
 手書きの手帳やExcelに近い、1日1時間の時間軸が明確な縦型（バーチカル）インターフェースを採用します。
 特にリズムを取り戻したい時期に、「時間の使い方」と「体調」を客観的に紐づけて確認しやすく、自身のパターン認識と改善に役立ちます。

## 機能候補

### MVPリリース時
1⃣ ユーザー機能
 ユーザー登録・ログイン機能。

2⃣ 記録UI
 スマホで見やすい縦型バーチカルカレンダー形式での、予定・記録のCRUD機能。

3⃣ 記録項目管理
 設定画面（マイページ）から以下の操作が可能。
 - 初期テンプレート（system項目）の表示・非表示切り替え。
 - ユーザー独自の記録項目（custom項目）の作成。
 - 作成した記録項目は、日記・体調記録フォームに自動で反映される。

4⃣ 日誌機能
 各記録日に対して、自由記述のメモ（日記）を記入・保存できる。

### 本リリース
5⃣ データ可視化
 - グラフ表示：睡眠時間や気分の推移など、記録したデータの週次・月次グラフ表示。
 - カレンダービュー：週間・月間カレンダーを表示するページを作る。

### 後のアップデート
6⃣ 報告書出力
 記録データを会社や医師に提出可能なPDF / CSV形式でのエクスポート機能。

7⃣ 継続サポート
 ユーザーが設定した時刻に記録を促すリマインド機能。

## 使用する技術スタック
- 使用するフレームワーク：Ruby on Rails 7.x
 学習した技術を最大限に活かすため。
- データベース：PostgreSQL
 本番環境のデプロイ先で推奨されることが多いため。
- デプロイ先：Render
 無料または安価で利用でき、Ruby on RailsとPostgreSQLの環境構築が容易なため。
- 使用予定のライブラリ
 Devise：ユーザー登録、ログイン、認証機能を迅速かつ安全に実装するため。
 Chartkick / Chart.js：記録データのグラフ化を容易に実現するため。
 Sidekiq：リマインド機能などの非同期処理を実現するため。
 Kaminari：カレンダーや記録一覧のページネーションを実装するため。
